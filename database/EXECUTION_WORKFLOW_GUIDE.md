# Compliance Execution Workflow Layer - Documentation

## Overview
The Compliance Execution Workflow Layer provides batch processing, subscription validation, and consolidated reporting on top of Compliance Engine v2.0.

## New Database Tables

### 1. compliance_sections
Master table for compliance sections
- section_code: MONTHLY, HALF_YEARLY, YEARLY, INCIDENT
- Seeded with 4 default sections
- Links forms to sections

### 2. compliance_execution_batches
Batch execution tracking
- tenant_id, branch_id (nullable)
- section_id, period_from, period_to
- status: Draft, Processing, Completed
- generated_report_path: Final consolidated report
- created_by: User who created batch

### 3. compliance_batch_forms
Individual form status within batch
- batch_id, form_id (unique constraint)
- status: Pending, AutoGenerated, Uploaded, NIL
- subscription_valid: Boolean flag
- requires_upload: Boolean flag
- completed_at: Timestamp

## Models Created

1. **ComplianceSection** - Section management
2. **ComplianceExecutionBatch** - Batch tracking with tenant scope
3. **ComplianceBatchForm** - Form-level status tracking

## Services Created

### ComplianceExecutionService

**Methods:**

1. `createBatch($tenantId, $sectionId, $periodFrom, $periodTo, $formIds, $branchId)`
   - Validates subscription per form
   - Creates batch and batch_forms entries
   - Sets requires_upload flag if subscription invalid

2. `processBatch($batchId)`
   - Iterates through all forms in batch
   - If upload_only: Status = Pending
   - If !subscription_valid: Status = Pending, requires_upload = true
   - Else: Calls ComplianceEngine::generateForm()
   - If no data: Status = NIL
   - Returns results array

3. `markFormAsUploaded($batchFormId, $filePath)`
   - Updates status to Uploaded
   - Sets completed_at timestamp

4. `getPendingForms($batchId)`
   - Returns forms requiring upload

### ComplianceReportBuilder

**Methods:**

1. `generateFinalReport($batchId)`
   - Fetches all batch forms
   - Collects generated PDFs
   - Collects uploaded files
   - Adds NIL sections
   - Adds "UPLOAD REQUIRED" alerts
   - Merges into consolidated PDF
   - Stores path in batch.generated_report_path

2. `mergePDFs($pdfPaths)`
   - Placeholder for PDF merging logic

## Controller Endpoints

### ComplianceExecutionController

**Routes:**
- GET `/compliance/execution` - Show UI
- GET `/compliance/sections` - Get all sections
- GET `/compliance/forms/{section}` - Get forms by section with subscription status
- POST `/compliance/batch/create` - Create new batch
- POST `/compliance/batch/{id}/process` - Process batch

## Workflow Steps

### Step 1: Create Batch
```php
POST /compliance/batch/create
{
    "section_id": 1,
    "period_from": "2024-01-01",
    "period_to": "2024-01-31",
    "form_ids": [1, 2, 3],
    "branch_id": 5
}
```

**Process:**
- Validates subscription for each form
- Creates compliance_execution_batches record
- Creates compliance_batch_forms for each form
- Sets subscription_valid and requires_upload flags

### Step 2: Process Batch
```php
POST /compliance/batch/{id}/process
```

**Process:**
For each form:
- If upload_only = true → Status = Pending
- If subscription_valid = false → Status = Pending, requires_upload = true
- Else → Generate form via ComplianceEngine
- If no data → Status = NIL
- Update batch status to Completed when all done

### Step 3: Upload Missing Forms (Manual)
```php
POST /compliance/batch/{id}/upload
{
    "batch_form_id": 123,
    "file": <uploaded_file>
}
```

### Step 4: Generate Consolidated Report
```php
POST /compliance/batch/{id}/report
```

**Process:**
- Collects all AutoGenerated PDFs
- Collects all Uploaded files
- Adds NIL sections with watermark
- Adds "UPLOAD REQUIRED" alerts for pending forms
- Merges into single PDF
- Stores path in batch.generated_report_path

### Step 5: Download Report
```php
GET /compliance/batch/{id}/download
```

## Subscription Validation Logic

```php
validateFormSubscription($tenantId, $form)
```

**Module Mapping:**
- Factories → workforce
- CLRA → clra
- Shops → shops
- EPF → epf
- ESI → esi

**Checks:**
- tenant_services.is_enabled = true
- athens_module_subscriptions (if exists)

**Result:**
- true → Auto-generate allowed
- false → requires_upload = true

## NIL Logic

**Triggers:**
- No data found in source tables
- Empty result from FormDataAggregator

**Actions:**
- Set status = NIL
- Add NIL watermark in report
- Mark as completed

## Alert Logic

**Triggers:**
- requires_upload = true AND status = Pending
- upload_only = true AND no attachment

**Actions:**
- Display "UPLOAD REQUIRED" in UI
- Add alert section in consolidated report
- Prevent batch completion

## Security Features

1. **Tenant Isolation** - Global scope on batches
2. **Ownership Verification** - Check tenant_id before processing
3. **Audit Trail** - created_by, IP tracking via ComplianceEngine
4. **Batch Locking** - Status = Completed prevents modification
5. **Authorization** - Middleware checks on all routes

## Usage Examples

### Create and Process Batch
```php
$service = app(ComplianceExecutionService::class);

$batch = $service->createBatch(
    tenantId: 1,
    sectionId: 1,
    periodFrom: '2024-01-01',
    periodTo: '2024-01-31',
    formIds: [1, 2, 3],
    branchId: 5
);

$results = $service->processBatch($batch->id);
```

### Generate Report
```php
$builder = app(ComplianceReportBuilder::class);
$filePath = $builder->generateFinalReport($batch->id);
```

### Dispatch Queue Job
```php
GenerateComplianceReport::dispatch($batch->id);
```
