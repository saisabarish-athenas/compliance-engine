## Database Schema

### compliance_sections
- 4 seeded sections: MONTHLY, HALF_YEARLY, YEARLY, INCIDENT
- Links to compliance_forms_master

### compliance_execution_batches
- tenant_id, branch_id (nullable)
- section_id, period_from, period_to
- status: Draft, Processing, Completed
- generated_report_path

### compliance_batch_forms
- batch_id, form_id (unique)
- status: Pending, AutoGenerated, Uploaded, NIL
- subscription_valid, requires_upload flags
- completed_at timestamp

## Workflow Summary

1. **Create Batch** - Select section, period, forms, branch
2. **Validate Subscriptions** - Check per form, set flags
3. **Process Batch** - Auto-generate or mark for upload
4. **Handle NIL** - Mark forms with no data as NIL
5. **Generate Report** - Consolidate all PDFs
6. **Download** - Get final report

## API Endpoints

- GET `/compliance/sections` - List sections
- GET `/compliance/forms/{section}` - Get forms with subscription status
- POST `/compliance/batch/create` - Create batch
- POST `/compliance/batch/{id}/process` - Process batch
- POST `/compliance/batch/{id}/report` - Generate report
- GET `/compliance/batch/{id}/download` - Download report

## Integration Points

### With Compliance Engine v2.0
- Uses ComplianceEngine::generateForm()
- Uses ComplianceEngine::validateFormSubscription()
- Stores in compliance_status and compliance_generation_logs

### With Subscription System
- Checks tenant_services table
- Maps act_type to module names
- Sets requires_upload if not subscribed

### With Branch System
- Uses branches table
- Filters compliance by branch_id
- Supports tenant-level (branch_id = null)
